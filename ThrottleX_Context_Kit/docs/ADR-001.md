# ADR-001 : Architecture ThrottleX

## Statut
Accepté

## Contexte
Service de rate limiting multi-tenant pour API REST.

## Décision
- **API** : FastAPI (async, OpenAPI auto)
- **Storage** : Redis (atomicité, TTL)
- **Algorithme** : Sliding Window + Token Bucket

## Conséquences
✅ Latence < 50ms
✅ Scalable horizontalement
⚠️ Dépendance Redis (SPOF potentiel)

## Alternatives Rejetées
- In-memory : pas distribué
- PostgreSQL : latence trop élevée
